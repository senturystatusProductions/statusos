# StatusOS UI Guidelines

## Product Feel

StatusOS should feel professional, focused, modern, and practical. It is a business operating system for music professionals, not a novelty chatbot.

## AI Command Center

The long-term AI layout should include:

- Specialist selector
- Main conversation area
- Current context
- Suggested actions
- Conversation history
- Tool results
- Clear status and error messages

## Interaction Rules

- Every button must have a clear purpose.
- Loading states must be visible.
- Destructive actions require confirmation.
- Errors must explain what happened without exposing technical secrets.
- Mobile layouts must remain usable.
- Keep existing StatusOS design language consistent.

## Accessibility

- Maintain readable contrast.
- Support keyboard navigation.
- Use visible focus states.
- Do not rely on color alone to communicate status.
